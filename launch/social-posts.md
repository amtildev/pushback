## LinkedIn Post

```text
Most AI assistants are optimized to be helpful.

That becomes a problem when "helpful" means agreeing with a weak premise.

I made Pushback, an open-source prompt and Codex skill that makes coding agents challenge assumptions before they act.

It asks the assistant to:

- give a verdict first
- identify the hidden assumption
- name the practical risk
- separate "this can be done" from "this should be done"
- offer a smaller or safer path

The goal is not negativity. The goal is better judgment before code, production changes, purchases, launches, or destructive operations.

Repo: https://github.com/amtildev/pushback
```

## Reddit Post

```text
Title: I made a small open-source prompt/skill to stop coding agents from agreeing too quickly

I kept running into the same issue with AI coding agents: they are very good at executing, but they often validate a weak premise before checking it.

So I made Pushback.

It is a tiny open-source Codex skill plus a universal copy-paste prompt. It makes the assistant give a verdict, identify the hidden assumption, name the risk, and suggest a better path before blindly implementing.

It is meant for product ideas, risky shell commands, architecture rewrites, production changes, and "is this actually a good idea?" moments.

Repo: https://github.com/amtildev/pushback
```
